  - eager_input_streaming is a newer Anthropic tool-use beta field (eager input
  streaming lets the model start emitting tool input deltas before the schema is
   fully validated). The provider you're hitting is rejecting it with a strict
  Pydantic-style "Extra inputs are not permitted" — meaning that endpoint hasn't
   been updated to know about the field.
  - The error is uniform across all 69 tools, which tells you opencode is
  injecting the flag into every tool definition at request-build time, not
  per-tool. So the fix lives in one place in opencode's request builder, not in
  each tool schema.
  - Common cause: opencode upgraded its anthropic SDK / added the
  eager-streaming beta header, but you're routing through a proxy/gateway
  (LiteLLM, OpenRouter, a self-hosted Anthropic-compatible shim, or a stale
  Bedrock/Vertex passthrough) that validates against an older schema.
  ─────────────────────────────────────────────────

  Diagnosis

  eager_input_streaming: true is being attached to every tool in the request.
  The endpoint receiving it doesn't accept that field. Two possibilities:

  1. You're not hitting api.anthropic.com directly — a proxy in the middle
  (LiteLLM, OpenRouter, custom gateway, Bedrock, Vertex) is doing strict schema
  validation and stripping/rejecting unknown fields. Anthropic's own API would
  accept it (or at least ignore it under the right beta header).
  2. Missing beta header — eager_input_streaming likely requires an
  anthropic-beta header to be sent alongside it. If opencode adds the field but
  not the header, even Anthropic's endpoint may reject it.
